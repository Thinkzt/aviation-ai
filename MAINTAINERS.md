# AviationAI 项目维护手册

## 团队分工

| 角色 | 负责人 | 职责 |
|------|--------|------|
| 主导 | 璇玑史CEO | 决策升级方向 |
| 开发 | 织锦 | 代码重构/GUI开发 |
| 民航专家 | 天枢 | 航务逻辑优化 |
| 数据官 | 磐石 | API封装/数据模型 |
| 进化引擎 | 薪火 | 版本迭代/发布 |

## 迭代计划

### 版本节奏
- **主版本**: 每季度一次 (v1.0, v2.0...)
- **小版本**: 每两周一次 (v0.1.2, v0.2.0...)
- **热修复**: 随时发布 (v0.1.x)

### 迭代内容
| 周期 | 内容 |
|------|------|
| 每日 | 代码提交 |
| 每周 | 功能测试 |
| 每两周 | 小版本发布 |
| 每月 | 大版本规划 |

## 代码规范
- 高效·精简·稳定·安全
- 单文件 < 500行
- 异常处理完善
- 禁止硬编码

## GitHub管理
- 分支: master (稳定) / develop (开发)
- Commit规范: feat/fix/docs/refactor
- 代码审查: 织锦 + 天枢
